# SuguDesk 開発チーム Notion ワークスペース設計

## 概要

SuguDesk開発チーム用のNotionワークスペース構成設計書。
MTG議事録・要件・アウトプットなどの情報を体系的にストックし、視認性高く、簡単に更新できることを目指す。

---

## トップレベル構成

```
📌 Dev Team Home（ダッシュボード）
│
├── 📋 MTG議事録
├── 📝 要件・仕様
├── 🚀 アウトプット
├── 🐛 課題・バグトラッカー
├── 📖 ナレッジベース
└── 🏥 クライアント別情報
```

---

## 1. Dev Team Home（ダッシュボード）

トップページ。チーム全員が最初に見るページ。

### 構成要素
- **リンクドビュー**: 各DBの直近アイテムを埋め込み表示
  - 今週のMTG予定（カレンダービュー）
  - 進行中の要件（ボードビュー、ステータス=実装中のみ）
  - 直近のアウトプット（テーブルビュー、最新5件）
  - 未解決の課題（ボードビュー、ステータス=対応中のみ）
- **クイックリンク**: よく使うページへのリンク集
  - Vercelダッシュボード
  - Google Sheetsマスター
  - Dify管理画面
  - GitHubリポジトリ

## 2. MTG議事録

### データベース設定

| プロパティ名 | 型 | 説明 |
|---|---|---|
| タイトル | タイトル | MTGの名称（例: 「第12回定例MTG」） |
| 日付 | 日付 | 開催日時 |
| 種別 | セレクト | `定例` / `臨時` / `クライアントMTG` / `レビュー` |
| 参加者 | マルチセレクト | メンバー名 |
| 関連クライアント | リレーション | クライアント別情報DB |
| ネクストアクション | チェックボックス | 未完了アクションがあるか |
| 関連要件 | リレーション | 要件・仕様DB |

### ビュー
- **カレンダービュー**（メイン）: 月間スケジュール俯瞰
- **テーブルビュー**: 全件一覧、検索・ソート用
- **ボードビュー**: 種別ごとにグループ化

### テンプレート

```markdown
## 議題
- [ ] 議題1
- [ ] 議題2

## 決定事項
-

## 議論メモ
-

## ネクストアクション
| 担当 | 内容 | 期限 |
|---|---|---|
|  |  |  |

## 関連リンク
-
```

## 3. 要件・仕様

### データベース設定

| プロパティ名 | 型 | 説明 |
|---|---|---|
| タイトル | タイトル | 要件名（例: 「きむら内科ダッシュボード追加」） |
| ステータス | セレクト | `アイデア` → `検討中` → `承認済` → `実装中` → `レビュー` → `完了` → `保留` |
| 優先度 | セレクト | `高` / `中` / `低` |
| 種別 | セレクト | `新機能` / `改善` / `バグ修正` / `技術的負債` |
| 担当者 | マルチセレクト | メンバー名 |
| 対象クライアント | リレーション | クライアント別情報DB（該当する場合） |
| 期限 | 日付 | 対応期限 |
| 関連MTG | リレーション | MTG議事録DB |
| 関連アウトプット | リレーション | アウトプットDB |

### ビュー
- **ボードビュー**（メイン）: ステータス別カンバン → 進捗が一目瞭然
- **テーブルビュー**: 全件一覧、フィルタ・ソート用
- **優先度ビュー**: 優先度でグループ化したボード

### テンプレート

```markdown
## 背景・目的
（なぜこの要件が必要か）

## 要件詳細
（何を実現するか）

## 技術的な検討事項
- 影響範囲:
- 依存関係:
- 対象ファイル:

## 受け入れ条件
- [ ] 条件1
- [ ] 条件2

## 参考資料
-
```

## リレーション設計（DB間の紐づけ）

```
MTG議事録 ←→ 要件・仕様
    │              │
    │              ↓
    │         アウトプット
    │              │
    ↓              ↓
クライアント別情報 ←── 課題・バグ
```

- **MTG議事録 ⇔ 要件・仕様**: MTGで議論した要件を紐づけ
- **要件・仕様 ⇔ アウトプット**: 要件がどのリリースで実現されたかを追跡
- **全DB → クライアント別情報**: クライアント軸で横断フィルタ可能

## 運用ルール

### 更新頻度
| セクション | 更新タイミング |
|---|---|
| MTG議事録 | MTG開催時（テンプレートから作成） |
| 要件・仕様 | 要件発生時、ステータス変更時 |
| アウトプット | リリース時 |
| 課題・バグ | 発見時即座に起票 |
| ナレッジベース | 新しい知見が得られた時 |
| クライアント別 | クライアント追加・設定変更時 |

### 命名規則
- MTG議事録: `YYYY-MM-DD 種別名`（例: `2026-04-10 定例MTG`）
- 要件: 動詞始まりで簡潔に（例: `ダッシュボードに時間帯別グラフを追加`）
- アウトプット: `vX.Y.Z リリース名`（例: `v1.3.0 きむら内科対応`）
- 課題: 現象を端的に（例: `ルナ: 14日間グラフでデータ欠損`）

### Tips
- Notionのテンプレートボタンを活用し、毎回同じフォーマットで記録する
- `@メンション`でタスクの担当者を明示する
- ページ内の見出しを統一し、目次（TOC）ブロックを先頭に置く
- Slackとの連携（Notion通知）でDB更新を自動通知する
