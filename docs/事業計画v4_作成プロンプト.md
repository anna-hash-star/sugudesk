# 事業計画v4 スプレッドシート作成指示書

## 概要
openpyxlでExcelファイル（事業計画v4.xlsx）を作成してください。
5シート構成、全セル関数（ベタ打ち禁止）、前提シートの値を変えれば全シート連動する設計にしてください。

## シート2: 月次事業計画

### 構造
- A列: ラベル
- B列〜AK列: M1〜M36（36ヶ月 = 3年）
- Row2: フェーズ名（IF文で前提のPhase区間から自動判定）
- Row3: M1, M2, ... M36

### 【営業活動】（Row5〜）
- Row5「月間新規SuguDesk契約数」: Phase判定して前提シート参照
  - `=IF(COLUMN()-1<=6, '前提'!$B$13, IF(COLUMN()-1<=12, '前提'!$B$14, IF(COLUMN()-1<=18, '前提'!$B$15, IF(COLUMN()-1<=24, '前提'!$B$16, '前提'!$B$17))))`

### 【顧客数】（Row7〜）
- Row7「SuguDesk期首顧客数」: B7=1（既存）, C7=B8, D7=C8...
- Row8「SuguDesk解約数」: =ROUNDDOWN(当月期首×'前提'!$B$8, 0)
- Row9「SuguDesk期末顧客数」: =期首+新規-解約
- Row10「BPO新規転換数」: =ROUNDDOWN(SuguDesk期末×'前提'!$B$10, 0) ※ただしM1-6は0（Phase1はBPO立ち上げ期）
- Row11「BPO解約数」: =ROUNDDOWN(BPO期首×'前提'!$B$9, 0)
- Row12「BPO期首顧客数」: B12=0, C12=B13...
- Row13「BPO期末顧客数」: =BPO期首+BPO新規-BPO解約

### 【売上】（Row15〜）
- Row15「SuguDesk月額売上」: =SuguDesk期末×'前提'!$B$4
- Row16「BPO事務長代行売上」: =BPO期末×'前提'!$B$6
- Row17「BPO採用/集患売上」: =BPO期末×'前提'!$B$7 ※M13以降のみ（IF文）
- Row18「導入費用」: =新規契約数×'前提'!$B$5
- Row19「MRR」: =Row15+Row16+Row17
- Row20「売上合計」: =Row19+Row18

### 【コスト】（Row22〜）
- Row22「CEO報酬」: ='前提'!$B$20
- Row23「エンジニア」: Phase1は ='前提'!$B$21*'前提'!$B$22、Phase2+は ='前提'!$B$21*'前提'!$B$23
  - `=IF(COLUMN()-1<=6, '前提'!$B$21*'前提'!$B$22, '前提'!$B$21*'前提'!$B$23)`
- Row24「BPO事務長」: ='前提'!$B$24 × Phase別人数（前提シートのBPO人員計画を参照）
  - `=IF(COLUMN()-1<=6, '前提'!$B$24*'前提'!$B$32, IF(COLUMN()-1<=12, '前提'!$B$24*'前提'!$B$33, ...))`
- Row25「BPOオペ」: 同様に ='前提'!$B$25 × Phase別オペ人数
- Row26「インフラ・ツール」: ='前提'!$B$26
- Row27「その他固定費」: ='前提'!$B$27
- Row28「LLM API変動費」: =(SuguDesk期末+BPO期末)×'前提'!$B$28
- Row29「営業/GTM」: ='前提'!$B$29
- Row30「コスト合計」: =SUM(Row22:Row29)

### 【損益】（Row32〜）
- Row32「営業利益」: =売上合計-コスト合計
- Row33「営業利益率」: =IF(売上合計<>0, 営業利益/売上合計, 0)
- Row34「累積損益」: B34=B32, C34=B34+C32, ...
- Row35「MRRベース損益」: =MRR-コスト合計

### 【キャッシュフロー】（Row37〜）
- Row37「調達資金」: B37に全調達額の合計（='前提'!B42+'前提'!B43+'前提'!B44+'前提'!B45）、他月は0
- Row38「Series A入金」: =IF(COLUMN()-1='前提'!$B$52, '前提'!$B$50, 0)
- Row39「公庫利息」: 据置期間中は ='前提'!$B$45*'前提'!$B$46/12、その後はPMT関数
- Row40「キャッシュ残高」: =前月残高+調達+SeriesA+売上合計-コスト合計-公庫返済
- Row41「ランウェイ（月）」: =IF(キャッシュ>0, IF(コスト>0, キャッシュ/コスト, ""), 0)

### 【人員数】（Row43〜）
- Row43「CEO」: 1（固定）
- Row44「エンジニア（換算）」: =IF(COLUMN()-1<=6, '前提'!$B$22/160, '前提'!$B$23/160)
- Row45「BPO事務長」: Phase別人数（前提参照）
- Row46「BPOオペ」: Phase別人数（前提参照）
- Row47「合計」: =SUM

## シート3: 年次事業計画

### 構造
- B列〜D列: 1年目, 2年目, 3年目
- すべて月次シートのSUM/参照

### 内容
- 【顧客数】期末顧客数: ='月次事業計画'!M列（12ヶ月目）, Y列（24ヶ月目）, AK列（36ヶ月目）
- 【売上】年間売上合計: =SUM('月次事業計画'!B20:M20)、=SUM(N20:Y20)、=SUM(Z20:AK20)
- 【売上内訳】SuguDesk/BPO事務長/BPO採用集患/導入費用 各行のSUM
- 【コスト】年間合計: =SUM(月次コスト合計の該当12ヶ月)
- 【コスト内訳】各費目のSUM
- 【損益】年間営業利益 = 売上 - コスト
- 営業利益率 = 利益/売上
- 累積損益 = 月次の期末累積損益を参照
- 【キャッシュ】期末キャッシュ残高、ランウェイ
- 【KPI】MRR（期末）、ARR（=MRR×12）
- 【人員】期末人数

## シート4: 資本政策表

### 【ラウンド別】（Row3〜）
| | 設立(B列) | シード(C列) | Series A(D列) |
|---|---|---|---|
| プレバリュー | - | ='前提'!B41 | ='前提'!B49 |
| エクイティ調達額 | ='前提'!B40 | ='前提'!B42+'前提'!B43+'前提'!B44 | ='前提'!B50 |
| デット（公庫） | - | ='前提'!B45 | - |
| ポストバリュー | - | =プレ+エクイティ | =プレ+調達額 |

### 【株式数】（Row10〜）
- シード株価: =プレバリュー/設立時株数 → ='前提'!B41/'前提'!B39
- VC新規株数: =ROUNDDOWN('前提'!B42/株価, 0)
- 佐藤 新規株数: =ROUNDDOWN('前提'!B43/株価, 0)
- 洋介 新規株数: =ROUNDDOWN('前提'!B44/株価, 0)
- シード後 発行済株数: =設立時株数+VC+佐藤+洋介
- Series A株価: ='前提'!B49/シード後株数
- Series A新規株数: =ROUNDDOWN('前提'!B50/SA株価, 0)
- SOPプール: =ROUNDDOWN((シード後+SA株数)×'前提'!B51/(1-'前提'!B51), 0)
- SA後 発行済株数: =シード後+SA+SOP

### 【Cap Table 持分比率】（Row22〜）
各ラウンド後の全株主の持分比率を関数で算出。
- CEO: ='前提'!B39/発行済株数
- Oikaze: =VC株数/発行済株数
- 佐藤: =佐藤株数/発行済株数
- 洋介: =洋介株数/発行済株数
- SOPプール: =SOP株数/発行済株数（Series Aのみ）
- Series A投資家: =SA株数/発行済株数
- 合計チェック: =SUM（1.0になること）

### 【感度分析】（Row35〜）
Series Aプレバリュー別（3億/5億/7億/10億）のFounder持分。
v3と同じ構造で、すべて関数。

### 【ARR逆算テーブル】（Row49〜）
プレバリューに必要なARRをマルチプル別（10x/15x/20x/30x）に表示。
='月次事業計画'の期末MRR×12と比較。

## 実装上の注意

1. **全セルを関数で**: 前提シート以外にベタ打ち数値を入れない。前提の値を変えたらすべて連動すること。
2. **シート間参照**: `='前提'!$B$20` のように絶対参照を使う。月次→年次は相対参照のSUM。
3. **Phase判定**: `IF(COLUMN()-1<=6, ...)` パターンで月番号からPhaseを判定する。B列=M1なのでCOLUMN()-1=月番号。
4. **書式設定**: 通貨は `#,##0`、比率は `0.0%`、ヘッダー行は太字＋背景色。損益のプラスは緑、マイナスは赤の条件付き書式。
5. **36ヶ月分**: B列（M1）〜AK列（M36）。
6. **初期値**: SuguDesk期首顧客数B7=1（既存顧客あり）、BPO期首B12=0。
7. **BPO採用/集患（Row17）はM13以降のみ**: `=IF(COLUMN()-1>=13, BPO期末×'前提'!$B$7, 0)`
8. **BPO新規転換（Row10）はM7以降のみ**: Phase1はBPO立ち上げ期で転換なし。`=IF(COLUMN()-1>=7, ROUNDDOWN(SuguDesk期末×'前提'!$B$10, 0), 0)`
